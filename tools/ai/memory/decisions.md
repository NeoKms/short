# Decisions

## 2026-07-13: базовая архитектура

- Выбраны Go, PostgreSQL и Redis.
- HTTP реализован стандартной библиотекой, чтобы сохранить малое число зависимостей.
- PostgreSQL хранит ссылки постоянно, Redis используется как ускоряющий cache-aside слой.
- Деплой повторяет `team-planning`: tag `v*` публикует образ для `linux/amd64`, `linux/arm64` и `linux/arm/v7` в GHCR и обновляет контейнер по SSH.
- PostgreSQL и Redis развёрнуты отдельно; API подключается к их опубликованным портам через `host.docker.internal`, сопоставленный с `host-gateway` при `docker run`.

## 2026-07-13: длинные исходные URL

- Максимальная длина `original_url` увеличена до 65535 символов, чтобы поддержать URL с закодированным состоянием приложения.
- SQL-миграции встраиваются и применяются в лексикографическом порядке; это позволяет добавлять миграции для уже развёрнутых баз данных.
- Стандартный HTTP 302 используется для URL длиной до 2048 байт. Более длинные назначения открываются через безопасно JSON-кодированную HTML-страницу с `window.location.replace`, поскольку nginx должен прочитать весь заголовок `Location` в одном proxy buffer и иначе возвращает 502.
- Валидация использует `url.Parse`, а не `url.ParseRequestURI`, чтобы URL fragment оставался fragment и не превращался в часть пути с `%23`.
