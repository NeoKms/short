# Agent instructions

## Язык

- Общайся с пользователем на русском.
- Код, команды, имена файлов, логи и идентификаторы оставляй в оригинальном виде.

## Перед началом задачи

Прочитай:

1. `tools/ai/memory/README.md`
2. `tools/ai/memory/project-context.md`
3. `tools/ai/memory/project-rules.md`
4. Релевантные файлы из `tools/ai/memory`.

## Работа с проектом

- Сначала ищи существующий паттерн через `rg`, затем вноси минимальные изменения в его стиле.
- Не добавляй зависимости без необходимости и не трогай несвязанные изменения.
- Схема PostgreSQL изменяется только новой SQL-миграцией в `internal/storage/migrations`.
- После изменений запускай `go test ./...`, `go vet ./...` и, если доступен Docker, сборку образа.
- После значимых архитектурных или эксплуатационных изменений обновляй `tools/ai/memory/project-context.md`, `decisions.md` и `change-log.md`.
- Не сохраняй секреты и реальные значения переменных окружения в репозитории или памяти агентов.
